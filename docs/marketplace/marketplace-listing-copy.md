# Marketplace Listing Copy

## Product Name

Sanfaani Schools

## Short Description

Laravel school management, CBT, result publishing, public result checking, licensing, guided onboarding, update preflight, and backup metadata foundation for single-school and managed deployments.

## Long Description

Sanfaani Schools is a Laravel-based school management and result access platform designed for schools that need student records, academic setup, result workflows, CBT support, scratch-card result access, role-based dashboards, public result checking, and commercial deployment foundations.

The package is prepared for single-school licensed buyers, direct sales, managed clients, reseller-assisted onboarding, white-label buyers where license terms allow, and demo/trial sales workflows. It includes safe foundations for installer guidance, license validation, guided onboarding, demo sessions, update package review, backup metadata, and restore planning.

Planned features are clearly marked in the documentation. This package does not include full billing automation, real marketplace API integration, automated restore execution, deployment automation, or real external update downloads.

## Key Features

- Super Admin and school workspaces.
- Student, class, subject, session, and term management.
- Teacher assignment and result submission workflow.
- Manual result entry, CSV upload, review, and publishing.
- CBT question banks, exams, attempts, and result publishing.
- Scratch-card request and public result checking workflow.
- Public school pages and result checker routes.
- Deployment mode and feature flag foundations.
- License activation and validation foundation.
- Role-based guided onboarding foundation.
- Demo request and demo credential foundation.
- Guided update package metadata and preflight foundation.
- Backup metadata, verification, retention, and restore-plan foundation.
- Documentation for installation, updates, backups, marketplace packaging, and support handover.

## Deployment Options

- Single-school marketplace buyer.
- Direct single-school licensed buyer.
- Managed client deployment.
- White-label buyer where permitted.
- Demo/trial sales environment.

## Buyer Requirements

- PHP 8.3 or newer.
- MySQL or MariaDB.
- Composer.
- Node.js and npm for frontend builds.
- Writable `storage` and `bootstrap/cache`.
- Hosting configuration that can point the web root to Laravel `public` where possible.

## Hosting Compatibility

The documentation includes cPanel and Namecheap shared-hosting guidance. Shell access is helpful but not assumed for backup and installer guidance.

## Installation Process

Buyers must upload the package, copy `.env.marketplace.example` to `.env`, configure hosting, domain, database, SMTP, and license values, generate the app key, run installation steps, run migrations after review, and link storage.

## Licensing Notes

The licensing foundation supports subscription, annual, lifetime, managed contract, white-label, trial, and demo license modes. Raw license keys are not stored. Marketplace license sync is planned and not implemented.

## Update Notes

The update manager stores package metadata, validates manifests, runs preflight checks, creates update logs, and prepares rollback metadata. It does not download external updates, extract code, run migrations, or apply patches from the browser.

## Backup Notes

The backup manager records backup metadata, verification status, retention state, logs, and manual restore plans. It does not run automated restore execution, expose backup downloads, or export `.env` secrets.

## Support Notes

Support should define installation assistance, response windows, customization boundaries, and paid managed-service terms before sale.

## What Is Included

- Laravel application source.
- Database migrations.
- Admin, school, CBT, result, public result checker, demo, onboarding, licensing, update, and backup foundations.
- Buyer-safe environment template.
- Marketplace packaging docs and checklists.
- Tests for package validation safety.

## What Is Not Included

- Production hosting setup by default.
- Domain, database, SMTP, or license configuration by default.
- Real marketplace API integration.
- Full billing/payment workflow.
- Deployment automation.
- Automated restore execution.

## Buyer Configuration Disclaimer

The buyer must configure hosting, domain, database, SMTP, storage permissions, queues/cron where needed, and license values. Never upload real `.env` files, secrets, logs, backups, local databases, `vendor`, `node_modules`, or private school data to a marketplace package.
