# Marketplace Documentation Checklist

Before submission, confirm the package includes:

- Product overview and requirements.
- Buyer installation checklist.
- Shared-hosting and cPanel guidance.
- Installer documentation: `docs/installation/single-school-installer.md`.
- Licensing documentation: `docs/licensing/license-activation.md`.
- Update documentation: `docs/updates/update-system-plan.md`.
- Backup documentation: `docs/backups/backup-system-plan.md`.
- Tenant isolation and security documentation.
- Changelog and release workflow.
- Support policy and customization boundaries.
- Marketplace listing copy.
- Screenshot checklist and demo checklist.
- Include/exclude manifest.
- Package validation checklist.

Docs must clearly mark planned features as planned and avoid claiming one-click deployment, full billing automation, real marketplace API sync, automated restore, or real external update downloads.
