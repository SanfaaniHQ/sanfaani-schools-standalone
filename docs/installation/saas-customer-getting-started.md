# SaaS Customer Getting Started

SaaS mode is the simplest path for normal school owners. You use Sanfaani Schools from your browser. You do not upload files, run commands, or install the application.

## What You Need

- A browser on a phone, tablet, or computer.
- Your school name and contact details.
- The login URL or demo link from Sanfaani.
- The owner or school admin login details.

You do not need Git, Composer, npm, SSH, cPanel, Laravel knowledge, database access, or terminal access.

## Getting Started

1. Request a demo or contact Sanfaani sales.
2. Confirm the school name, contact person, phone number, email, and preferred onboarding path.
3. Open the link Sanfaani sends you in your browser.
4. Sign in to your school workspace.
5. Complete guided onboarding.
6. Add the school profile, sessions, terms, classes, subjects, staff, and students.
7. Start result setup, result entry or upload, publishing review, and parent result access.

## Demo Request Next Steps

After you request a demo, Sanfaani may prepare a demo workspace or contact you to schedule a walkthrough. Demo credentials are temporary and should not be used for live school records.

## First School Workspace Checklist

- Confirm school name, logo, address, email, and phone.
- Add the current academic session and term.
- Add classes and subjects.
- Add staff users who will work on results.
- Add or upload students.
- Configure grading and result settings.
- Publish only reviewed results.

## When To Ask For Help

Ask Sanfaani for help if you are unsure about setup, student upload templates, result workflow, scratch-card access, or parent result checking. SaaS customers should not need to ask a hosting provider for Git, Composer, npm, or database setup.
