# Managed Client Handover Guide

Managed mode is for premium clients where Sanfaani installs, configures, verifies, and hands over the system.

## What Sanfaani Handles

The Sanfaani team handles setup according to the managed agreement. This can include hosting review, domain setup guidance, application upload or deployment, environment configuration, database configuration, migrations, storage checks, SMTP configuration, owner admin creation, school profile setup, license validation, and launch verification.

## What The Client Provides

- School name and preferred display name.
- Owner or primary administrator name.
- Administrator email and phone number.
- Logo and branding details where available.
- Domain or subdomain details.
- Hosting access boundaries if the client owns hosting.
- SMTP or email-provider details where the client owns email.
- Required staff roles for the first handover.

## Handover Packet

The handover should include:

- Login URL.
- School workspace name.
- Owner admin email.
- Secure password delivery method.
- Support contact and escalation route.
- Backup responsibility.
- Update responsibility.
- Hosting owner and access boundaries.
- License mode and deployment mode.
- Known limitations or planned follow-up tasks.

## Client Verification

Before signoff, the client should:

1. Open the login URL in a browser.
2. Sign in with the provided account.
3. Confirm the school name and profile details.
4. Confirm current session and term setup.
5. Confirm staff accounts or request changes.
6. Confirm parent-facing result access links where enabled.
7. Confirm who to contact for support.

## Boundaries

Managed setup is service-backed. Clients should not run `/install`, Git, Composer, npm, or terminal commands unless Sanfaani specifically asks a technical contact to assist. Managed updates, backups, and restore execution follow the signed support agreement.
