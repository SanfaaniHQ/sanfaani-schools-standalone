# Branding Setup Guide

Use the Branding screen to configure the visible identity for the current deployment.

Recommended order:

1. Set the display brand name.
2. Upload a PNG, JPG, WEBP, or ICO logo.
3. Upload a favicon.
4. Configure primary, secondary, and accent colors as six-character hex values.
5. Review login, dashboard, email footer, and report footer text.
6. Confirm the preview does not expose private file paths.

Deployment behavior:

- SaaS: Super Admin manages platform branding.
- Single-school: Super Admin can configure local guided branding, while school admins can configure school branding.
- Managed: Super Admin manages managed client branding.
- White-label: White-label controls remain locked unless feature and entitlement checks pass.
