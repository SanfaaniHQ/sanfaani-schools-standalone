# Demo Readiness Checklist

Use this checklist before a direct sales demo, marketplace demo, reseller demo, or managed-client walkthrough.

## Demo Safety

- [ ] Demo database uses fake or seeded demo data only.
- [ ] No real student, parent, staff, payment, license, email, backup, or production data is present.
- [ ] Demo users and roles are approved for public or controlled demo use.
- [ ] No real credentials are included in docs, screenshots, chat, or slides.
- [ ] Destructive actions are disabled or avoided in the demo flow.
- [ ] Known limitations are ready to explain in plain language.

## Access And Branding

- [ ] Admin login is tested.
- [ ] School Admin login is tested.
- [ ] Demo school branding is configured.
- [ ] Logo, favicon, colors, login wording, dashboard heading, email footer, and report footer are safe.
- [ ] Support contact text is demo-safe.

## School Setup

- [ ] Sample school profile exists.
- [ ] Sample students exist.
- [ ] Sample classes, subjects, session, and term exist.
- [ ] Staff roles are ready for walkthrough.
- [ ] Permissions are configured for the demo path.

## Module Samples

- [ ] Admissions form is ready.
- [ ] Sample admission application exists.
- [ ] Attendance sample exists.
- [ ] Attendance report sample exists.
- [ ] Offline attendance boundary is explained as attendance-focused browser capture/sync.
- [ ] Finance sample exists with fee item, assignment, invoice, payment, report, and audit example.
- [ ] LMS sample classroom/material exists.
- [ ] CBT sample exists and is safely linked where needed.
- [ ] Live class sample exists with a manual meeting link.
- [ ] Communication templates exist.
- [ ] Notification logs contain safe demo events.
- [ ] Reports Center has useful sample data.

## Operations Readiness

- [ ] Backup status is ready to explain.
- [ ] Installer status is ready to explain.
- [ ] License status is ready to explain with masked values only.
- [ ] Update status/preflight boundary is ready to explain.
- [ ] System health status is ready to explain.
- [ ] Support runbooks and release handoff docs are ready to show.

## Demo Talk Track Boundaries

- [ ] Do not claim full offline system support.
- [ ] Do not claim SaaS billing or payment gateway automation.
- [ ] Do not claim online update server, auto-download, or destructive auto-apply.
- [ ] Do not claim remote license server behavior.
- [ ] Do not claim real Google Meet, Zoom, or Microsoft Teams API automation.
- [ ] Do not claim Next.js website integration as completed.
- [ ] Do not claim dependency vulnerabilities are fixed.
