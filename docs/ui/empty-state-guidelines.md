# Empty State Guidelines

Empty states should explain what is missing without turning the page into documentation.

## Usage

- Use for empty queues, no records, first setup, and filtered results with no matches.
- Keep titles short.
- Keep body copy operational and role-aware.
- Include one action only when there is a clear next step.

## Tone

Empty states should be calm and neutral. Use alerts for warnings, errors, and production safety messages.
